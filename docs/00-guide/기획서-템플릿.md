# Claude Sticky 기획서 템플릿

## Context

Claude Sticky 프로젝트에 기능을 추가하거나 개선할 때, 사용자가 작성한 기획서를 Claude가 읽고
PDCA 사이클(Plan → Design → Do → Check)로 구현할 수 있도록 표준 형식을 정의합니다.

---

## 프로젝트 현황 참고 (Claude용)

Claude가 기획서를 받을 때 항상 참고해야 할 현재 프로젝트 구조:

### 페이지/패널
| 패널 ID | 컴포넌트 파일 | 역할 |
|---------|-------------|------|
| `tasks` | `compact-tasks-panel.tsx` | 과제 보드 (상태별 탭, 정렬/검색/필터) |
| `mypage` | `compact-mypage-panel.tsx` | 마이페이지 (포인트, 레벨, 배지) |
| `league` | `compact-league-panel.tsx` | 리더보드 |
| `shop` | `compact-shop-panel.tsx` | 상점 (포인트로 아이템 구매) |
| `settings` | `compact-settings-panel.tsx` | 설정 |
| `admin` | `compact-admin-panel.tsx` | 관리자 패널 (팀원 관리, 초대 코드) |
| `team-setup` | `compact-team-setup-panel.tsx` | 팀 초기 설정 |
| `notifications` | `compact-notifications-panel.tsx` | 알림 |
| `notion` | `compact-notion-panel.tsx` | Notion 연동 (어드민 전용) |

### DB 테이블 (Supabase)
| 테이블 | 주요 컬럼 | 용도 |
|--------|----------|------|
| `teams` | id, name, invite_code | 팀 정보 |
| `profiles` | id, nickname, role(admin/member), team_id, points, level | 사용자 |
| `tasks` | id, title, description, status, priority, points, due_date, assigned_to[], team_id, notion_id | 과제 |
| `task_comments` | id, task_id, user_id, content | 과제 댓글 |
| `shop_items` | id, name, description, price, team_id | 상점 아이템 |
| `purchases` | id, user_id, item_id | 구매 내역 |
| `point_logs` | id, user_id, amount, reason, task_id | 포인트 내역 |
| `notifications` | id, user_id, title, body, read, type | 알림 |

### 스토어 (Zustand)
| 스토어 | 파일 | 관리 데이터 |
|--------|------|------------|
| `useAuthStore` | `auth-store.ts` | user(profile), session |
| `usePanelStore` | `panel-store.ts` | activePanel, panelState |
| `useTaskStore` | `task-store.ts` | tasks[], filters, sorting |
| `useNotificationStore` | `notification-store.ts` | notifications[], unread count |

### 권한 구분
- `admin`: 과제 생성/수정/삭제, Notion 연동, 팀원 관리, 초대 코드 발급
- `member`: 과제 상태 변경, 댓글 작성, 상점 구매

### 기술스택
- Next.js 16 App Router + TypeScript + Tailwind CSS v4 + shadcn/ui
- Zustand v5 (상태관리) + React Hook Form v7 + Zod v4 (폼 검증)
- Supabase (DB + Auth + Realtime) + @notionhq/client v5
- 패널 크기: 360px 고정 (드래그앤드롭 미지원)
- 가상 사무실: Canvas API (virtual-office.tsx)

## 기획서 템플릿

아래를 복사해서 기능별 기획서를 작성하세요.

---

```markdown
# 기획서: {기능명}

## 1. 기능 개요

**기능명**: (예: 과제 댓글 기능, 포인트 자동 지급, 팀원 초대 UI 개선)
**목적**: 이 기능이 왜 필요한가? 어떤 문제를 해결하는가?
**대상 사용자**: admin / member / 모두

## 2. 영향 범위

**관련 패널**: (예: tasks, admin, mypage 등)
**관련 DB 테이블**: (예: tasks, profiles, point_logs 등)
**신규 테이블 필요**: 있음 / 없음 (있다면 컬럼 설명)
**관련 스토어**: (예: useTaskStore, useAuthStore 등)
**API 엔드포인트 필요**: 있음 / 없음 (있다면 경로 및 역할)
**환경변수 필요**: 있음 / 없음 (있다면 키 이름)

## 3. 구현 요구사항

### Must Have (반드시 포함)
- [ ] 요구사항 1
- [ ] 요구사항 2
- [ ] 요구사항 3

### Nice to Have (있으면 좋음)
- [ ] 선택 사항 1
- [ ] 선택 사항 2

### 제외 (Out of Scope)
- 이번 기획에서 구현하지 않을 사항

## 4. UX 흐름 / 화면 설명

### 흐름
1. 사용자가 {A}를 한다
2. {B} 화면이 나타난다
3. 사용자가 {C}를 입력/선택한다
4. {D} 결과가 표시된다

### 주요 UI 요소
- **{컴포넌트명}**: 역할 및 위치 설명
- **{버튼/폼 필드}**: 표시 조건 및 동작

### 에러/예외 처리
- {상황}: {처리 방법} (예: 토스트 메시지 표시)

## 5. 수용 기준 (Acceptance Criteria)

테스트 가능한 형태로 작성하세요.

- [ ] {기능}을 하면 {결과}가 표시된다
- [ ] {조건}일 때 {제약}이 적용된다
- [ ] admin만 {기능}에 접근할 수 있다
- [ ] {엣지 케이스}에서 오류 없이 동작한다

## 6. 참고 / 제약사항

- 패널 크기 360px 제약: 복잡한 레이아웃 지양
- Realtime 필요 여부: 있음 / 없음 (Supabase Realtime 구독)
- 기타 기술적 제약이나 참고할 기존 구현:
  - (예: compact-tasks-panel.tsx의 TaskDialog 패턴 참고)
  - (예: supabase/schema.sql에 컬럼 추가 필요)
```

## 작성 예시

아래는 "과제 댓글 기능" 기획서 예시입니다.

```markdown
# 기획서: 과제 댓글 기능

## 1. 기능 개요

**기능명**: 과제 댓글 작성 및 조회
**목적**: 과제 상세 화면에서 팀원 간 소통을 가능하게 해 별도 메신저 없이 과제 맥락을 유지
**대상 사용자**: 모두 (읽기), 팀원 본인 (작성/삭제)

## 2. 영향 범위

**관련 패널**: tasks
**관련 DB 테이블**: tasks, task_comments, profiles
**신규 테이블 필요**: 없음 (task_comments 테이블 이미 존재)
**관련 스토어**: useTaskStore (댓글 목록 추가)
**API 엔드포인트 필요**: 없음 (Supabase 클라이언트 직접 사용)
**환경변수 필요**: 없음

## 3. 구현 요구사항

### Must Have
- [ ] 과제 상세 시트(TaskDetailSheet)에 댓글 목록 표시
- [ ] 댓글 입력창 + 전송 버튼 (Enter 키 지원)
- [ ] 내가 쓴 댓글만 삭제 가능 (admin은 전체 삭제 가능)
- [ ] 댓글 작성 시 Realtime으로 즉시 반영

### Nice to Have
- [ ] 댓글 수정 기능
- [ ] 댓글에 이모지 반응

### 제외
- 댓글 알림 기능 (별도 기획으로 분리)
- 멘션(@팀원) 기능

## 4. UX 흐름

1. 과제 카드 클릭 → TaskDetailSheet 열림
2. 하단 댓글 섹션에 기존 댓글 목록 표시 (최신순)
3. 입력창에 내용 작성 후 Enter 또는 전송 버튼 클릭
4. 댓글이 목록 최하단에 즉시 추가됨
5. 내 댓글 우클릭 또는 삭제 아이콘으로 삭제

### 주요 UI 요소
- **댓글 목록**: 아바타 + 닉네임 + 시간 + 내용
- **입력창**: placeholder "댓글을 입력하세요..." + 전송 버튼

### 에러 처리
- 빈 댓글 전송 시도: 버튼 비활성화로 방지
- 전송 실패: "댓글 전송에 실패했습니다" 토스트

## 5. 수용 기준

- [ ] 댓글 입력 후 Enter → 목록에 즉시 표시된다
- [ ] 빈 댓글은 전송되지 않는다
- [ ] 다른 팀원이 댓글 추가 시 새로고침 없이 내 화면에 표시된다
- [ ] 내 댓글만 삭제 버튼이 표시된다 (admin은 전체 표시)
- [ ] 삭제 확인 없이 즉시 삭제된다 (단순 삭제)

## 6. 참고

- `task_comments` 테이블: id, task_id, user_id, content, created_at
- `task-detail-sheet.tsx` 파일에 댓글 섹션 추가
- Supabase Realtime: `task_comments` 테이블 구독 추가 필요
```

## 기획서 제출 방법

1. 위 템플릿을 복사해서 기능 내용 작성
2. Claude에게 다음 형식으로 전달:

```
/pdca plan {기능명-kebab-case}

아래 기획서를 기반으로 Plan 문서를 작성해주세요:

{기획서 내용 붙여넣기}
```

3. Claude가 PDCA Plan → Design → Do → Check 사이클로 구현합니다.
